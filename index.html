No Context Switching
All three domains — navigation, intelligence, and execution — are always on screen. Developers never lose their place or their active agent run while browsing.
Left-to-Right Flow
The layout encodes the workflow: Browse → Understand → Act → Observe. Each panel naturally preconditions the next decision without instruction.
Execution Stays Mounted
The right panel persists through long-running AI tasks. Switching repos or scrolling never hides an active run.

Three-Layer Health Cards
Each card shows: (1) the status value, (2) contextual detail ("Last scan 2 days ago"), and (3) an operational signal ("Safe to run automation"). A raw number becomes a risk assessment.
Ordered by Urgency
The intelligence feed sorts findings by severity — failing tests surface first, stale scans second. Developers see the critical issue first without scanning the whole list.
Plain English, Not System Data
"3 outdated dependencies detected — Patch-level updates are available and safe to review." Every finding pairs a title with an implication. No raw numbers, no jargon.

Five Tasks — No Overload
Each task has a short description and a “Best For” label, so the developer can decide quickly without reading extra documentation.
Repo Context in the Header
The task modal keeps design-system-core visible, so the developer always knows which repository the agent will act on.
Signal-Driven, Not Static
The recommendation changes based on repository health. If tests fail, it recommends tests. If security is stale, it recommends a scan.
Context Explains Why
The recommendation includes a short context line so the agent feels explainable, not like a black-box action button.

Failure is Explained

Context-Preserving Retry
The Retry button appears at the failure point. Execution context is preserved — the agent restarts cleanly. Failure feels fixable, not final.
Partial Signal Captured
"Preserved partial logs for inspection." Even failed runs surface partial findings. Partial signal is more valuable than silence.
Confidence Scoring
Low confidence after a failure explicitly communicates how much to trust the partial result. Developers calibrate their next action accordingly.
07
Visual Design System
Apple-inspired enterprise minimalism. Every color, radius, and typographic choice is functional — nothing is decorative.
SEMANTIC COLOR — 5 ROLES
Indigo — Primary
Actions, links, selected state, AI features
Green — Success
Passing tests, healthy repos, completion
Amber — Warning
Stale scans, moderate risk, review needed
Red — Danger
Failing tests, high risk, execution failures
Cool Neutral
oklch-tinted — 255–265° hue for visual cohesion
Color is Never Decorative
Each color role is exclusive. Green appears only for success, red only for danger. This makes status scannable in under a second — no label reading required.
